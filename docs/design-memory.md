# Project Design Memory

This project follows a single visual direction and all frontend work should stay aligned with it.

## Creative north star

- The product should feel like "The Chromatic Curator".
- The UI should avoid sterile productivity-app defaults.
- The experience should combine editorial boldness with modular Bento structure.

## Non-negotiable visual rules

- No 1px borders as the primary way to separate sections.
- Structure should come from tonal layering, spacing, and surface shifts.
- Corners must feel intentionally rounded; small-radius defaults do not fit this project.
- Static surfaces should not rely on heavy shadows.
- Whitespace should feel generous and premium.

## Surface model

- Base shell uses a clean neutral canvas.
- Primary sections use a slightly darker container surface.
- Cards that need emphasis should sit on the brightest surface.
- Floating states can use glassmorphism with blur.

## Color direction

- Deep blue is the main primary anchor.
- Energetic orange and forest green are core accent colors.
- Secondary text should keep a cool-toned, intentional feel instead of generic gray.
- Primary actions can use subtle gradients to avoid flat-looking CTAs.

## Typography direction

- Headlines and display moments should feel editorial and geometric.
- Body text and metadata should stay highly legible and compact.
- Card titles must dominate over metadata.

## Component guidance

- Buttons should use pill shapes to contrast with larger rounded containers.
- Chips and tags should carry strong color.
- Kanban cards should use a left intent bar for status or priority.
- Inputs should be borderless by default, with a soft focus treatment.
- Sidebar navigation should feel like a slim Bento tile with rounded active pods.

## Product fit for this project

For this Kanban app specifically:

- The board should feel curated, not corporate.
- Project cards should read like premium tiles with clear hierarchy.
- Tags, state, and activity should be visually bold and easy to scan.
- Any new screen should be evaluated against this design memory before implementation.
